# GEMINI - .NET MAUI 애플리케이션 개발 가이드

이 문서는 .NET MAUI를 사용하여 효율적이고 유지보수가 용이한 모바일 애플리케이션을 개발하기 위한 핵심 원칙과 프로세스를 정의합니다.

## 1. 핵심 개발 원칙

*   **유지보수성 (Maintainability):** 모든 코드는 미래의 내가 쉽게 이해하고 수정할 수 있도록 작성합니다. 기능 변경 및 확장이 용이한 구조를 최우선으로 합니다.
*   **모듈화 (Modularity):** 기능, 화면, 데이터 등 애플리케이션의 구성 요소를 독립적인 단위(클래스, 파일)로 명확하게 분리합니다.
*   **가독성 (Readability):** 일관된 코딩 스타일과 명확한 네이밍 규칙을 사용하여 누구나 쉽게 코드를 읽고 이해할 수 있도록 합니다.

## 2. 아키텍처: MVVM (Model-View-ViewModel)

.NET MAUI 개발의 표준 패턴인 MVVM을 채택하여 UI와 비즈니스 로직을 분리합니다.

*   **Model:** 애플리케이션의 데이터와 핵심 비즈니스 로직을 담당합니다. (예: `User.cs`, `Product.cs`)
*   **View:** 사용자에게 보여지는 UI(화면)입니다. XAML로 정의되며, 자체적인 로직은 최소화합니다. (예: `LoginPage.xaml`)
*   **ViewModel:** View와 Model을 연결하는 중재자입니다. View에 필요한 데이터(속성)와 기능(명령)을 제공하며, View의 상태를 관리합니다. (예: `LoginPageViewModel.cs`)

이 구조는 각 부분이 독립적으로 개발되고 테스트될 수 있게 하여 유지보수성을 극대화합니다.

## 3. 권장 폴더 구조

프로젝트의 파일을 체계적으로 관리하기 위해 다음과 같은 폴더 구조를 권장합니다.

```
/ (Project Root)
|
|- Views/         # UI 페이지 (XAML 파일)
|- ViewModels/    # View와 연결될 ViewModel 클래스
|- Models/        # 데이터 모델 클래스
|- Services/      # API 연동, 데이터베이스 접근 등 서비스 로직
|- Converters/    # 데이터 바인딩 시 값 변환을 위한 클래스
|- Resources/     # 이미지, 폰트, 스타일 등
```

## 4. 개발 프로세스

1.  **요구사항 분석:** 구현할 기능 또는 화면을 명확히 정의합니다.
2.  **Model 정의:** 해당 기능에 필요한 데이터 구조(Model)를 설계합니다.
3.  **ViewModel 구현:** View에 필요한 데이터와 명령을 포함하는 ViewModel을 작성합니다.
4.  **View 구현:** XAML을 사용하여 화면(View)을 구성하고 ViewModel에 바인딩합니다.
5.  **연결 및 테스트:** View, ViewModel, Model이 올바르게 상호작용하는지 확인합니다.

## 5. 변경사항 추적 및 로그

개발 과정의 변화와 요구사항의 흐름을 추적하기 위해 `development_log.txt` 파일을 생성하고 관리합니다.

*   **로그 시점:** 기능 추가/수정/삭제, 아키텍처 변경, 주요 버그 수정 등 의미 있는 변경이 완료될 때마다 기록합니다.
*   **로그 형식:**

    ```
    # YYYY-MM-DD
    
    ## [변경 유형] - 변경 내용 요약
    
    ### 상세 설명
    - (선택 사항) 변경의 이유, 구현 방식, 참고 사항 등을 상세히 기술합니다.
    
    ---
    ```

*   **변경 유형:** `[ADD]`, `[FIX]`, `[UPDATE]`, `[REFACTOR]`, `[DOCS]` 등

---
